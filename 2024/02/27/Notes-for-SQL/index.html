Cls01

数据

数据库

在计算机内部，特点：永久存储、有组织，可共享

数据库管理系统（DBMS）

->数据组织管理

->提供DML语言用以操纵数据

->事务管理，保证安全性和完整性

数据库系统

一般由数据库，数据库管理系统、应用系统，数据库管理员构成

Cls02

数据模型

对现实世界数据特征的抽象。

  1. 概念模型：用于数据库的设计（E-R图）
  2. 逻辑模型和物理模型：
    • 逻辑模型是按照计算机系统的观点对数据进行建模
    • 物理模型是对数据最低层的抽象，主要是面向计算机硬件的

数据模型的组成要素

一、数据结构

描述数据库的组成对象以及对象之间的联系，是对系统静态特性的描述。

二、数据操作

是对系统动态特性的描述。

三、数据库的完整性约束条件

  • 空值
  • 实体完整性
  • 参照关键字
  • 企业约束

概念模型中的名词

  1. entity：客观存在并可以相互区别的事物。
  2. attribute：实体具有的特性
  3. key：唯一标识实体的属性集称为key
  4. domain：一组具有相同数据类型的值的集合，即属性的取值范围
  5. entity type：具有相同属性的实体必然具有共同的特征，用实体名或属性名刻画同类实体
  6. entity set：同一类型实体的集合称为实体集
  7. relationship：1:1,1:n,m:n

数据模型中的关系模型

  • relation
  • tuple：行
  • attribute：列
  • key：only attribute
  • domain：range

Cls03

数据库的设计包括三个步骤：

  1. 概念模型（ER建模）

    • Strong Entity Type
    • Weak Entity Type
  2. 逻辑模型（规范化）
  3. 物理模型

概念模型转逻辑模型的一些特点

能出现的所有模式：(1,1);(0,1);(1,n);(0,n);

  • 强实体支配弱实体，强实体的主关键字进弱实体做主关键字和外部关键字(限定弱实体)
  • 一对多的情况，一定是一支配多，支配方的关键字进入被支配方做外部关键字（无论多的一方是弱还是强）
  • 一方强制的一对一关系，一定是1，1支配0，1
  • 强弱双方都是强制的一对一关系，需要合并表格（1，1：1，1），不是强弱的需要考虑支配角色
  • 在非强制的一对一关系（0，1：0，1），让受限制较少的一方支配（进入）受限制较多的一方，做外部关键字（通常受限较少的一方数量更多）
  • (0,n)-(0,1):当联系有属性的时候需要把属性放入多的一方。
