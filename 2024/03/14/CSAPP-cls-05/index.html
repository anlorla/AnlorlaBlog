立即数是一个常数，mov 指令不允许从Mem到Mem，并且让立即数作为目标地址是没有意义的。

$表示立即数，%表示寄存器，如果加上括号就是内存地址。

除此之外，在mov时可以添加偏移量，体现在括号外面的数字上。

一些例子

Arithmetic & logical operations

关于lea指令：（load effective address）

从引用形式来看，lea指令似乎从内存读数据到寄存器，但实际上lea指令并不涉及内存的引用，lea指令能够执行有限指令的乘法和加法（线性组合？）其中，Dst只能是寄存器，sal是左移运算 。

关于sar和shr，由于右移有两种情况，前者是算数右移，后者是逻辑右移。算数右移左侧需要补1，但二者在位运算上是一样的。

Control

程序不止是顺序执行的，我们会遇到条件和循环语句等等。

condition code

需要注意的是，leaq指令不改变任何条件码，因为它是用来进行地址计算的。

SET指令

  1. 可以根据条件码的某种组合将一个字节设置为0or1
  2. 可以条件跳转到程序的其他部分
  3. 可以有条件的传送数据

SET指令的不同后缀表示了条件码的不同组合。

JUMP指令

基于条件数据传送(conditional move)的代码比基于条件控制转移(conditional jump)的代码性能要好，前者计算一个条件操作的两种结果，然后在两种结果中选择其中一个作为结果。

一个不严谨但有用的理解：

  • 基于条件控制转移：在分岔口走错路只能回去重走
  • 基于条件数据传送：在分岔口分身一起走，走错了就保留另外一个分身

但是也有bad cases for conditional move,如果有空指针引用将会造成无法预知的影响,所以它的应用是在安全限制下才能进行的。

Loop

关于循环，所有的循环都可以使用do-while来表示，也就可以使用goto来表示，从某种意义上来说goto的特性非常接近汇编语言的特性。

Switch

使用跳转表是一种非常有效的实现多重分支的方法，有它的存在，switch语句中获取所需位置的时间复杂度是O(1)

  • 当范围不在case的索引时会进行bias的操作，保证无论最小值是多少都会通过增加偏置值的方式变成0
  • 如果case值比较稀疏，用到的值都会被变成if-else代码,编译器使用类似于二分的操作去索引
