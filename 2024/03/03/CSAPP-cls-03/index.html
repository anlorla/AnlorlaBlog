CSAPP-cls-03

unsigned integer & 2’s complement 的运算

  • UMax UMin
  • Tmax TMin
  • 补码运算中符号位参与运算

加法

在舍弃进位的情况下，unsigned int的加法和2‘s ccomplement 的加法是模运算下的加法。

乘法

在舍弃进位的情况下,unsigned int 的乘法运算等同于看作补码时补码的乘法。

eg.1: (unsigned int)13x14=(2’s complement) -3x-2

unsigned int 的移位运算

  • 乘以2相当于左移一位，其实是增加每一位的权重，除法同理。

  • Most machines shift and add faster than multiply.

  • 当被除数不能被2整除时的移位运算解释了为什么正整数的除法是向下取整的。
  • 无符号数的移位只能是逻辑移位。

2’s complement 的移位运算

  • 左移在右侧添0
  • 补码的除法运算有两种移位运算：逻辑移位和算术移位。
  • 在进行算术右移时，需要在左侧补1，在某些时候还需要添加偏移量，如-3/2

补码计算相反数vs原码计算补码

将补码变成反码后再加一可以得到相反数的补码，注意和计算原数补码的方式区分开。（除去符号位，取反加一得补码）

TMin的相反数是？ 还是TMin!

在使用unsigned 时可能会遇到的tricks

Byte

  • 有32 bit的机器和64 bit的机器，也即4字节和8字节的区别
  • 关于byte ordering:有大端和小端两种方式，即最低有效字节在机器中是放在末位还是首位，直观来看也即地址是递增还是递减

Some Quizs

  • 第三题是掩码运算，保证了后三位是1，所以左移30位后是11000……00,是负数
  • ux>-1会隐式转换
  • 第五题，如果y=TMin时不成立
  • 倒数第二题，由于不对称性，x=TMin时，相反数不能表示，还是TMin
  • 最后一题，如果x!=0的话是正确的
