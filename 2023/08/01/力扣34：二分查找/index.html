力扣第34题：二分查找法

发表于2023-08-01|更新于2023-12-04|Leetcode刷题日记
|阅读量:

二分查找刷题记录

一个月前学的二分查找又写不对了，赶紧来记录一下易错知识点。

一个理解的关键点：二分的本质不是有序，而是其一分为二的性质！实际上，使用二分时我们往往是使用一个性质将一组数一分为二，即一组数据被这个性质划分成两种状态（true or false），二分就是用来寻找这个性质的边界的。

知识点

第一种：左闭右闭

  • while的括号中应该取等于，因为left=right时这个边界是有意义的（参考Carl）
  • [left,right]中每一次循环左右边界的值都会进入操作，如果当前的nums[middle]不是target，说明当更新边界的时候就不用在索引为middle的地方停留了，即right=middle-1 或者 left=middle+1

第二种：左闭右开

  • while的括号中不应该取等于，因为left=right时边界无意义
  • 如果当前的nums[middle]不是target，更新边界的时候应该在索引为middle的地方停留，如果是right=middle-1就是让索引为middle-1的元素跳过了检查，遗漏了一些元素。注意，左边界是闭，所以仍然是middle+1

题目练习（力扣 34）

34. 在排序数组中查找元素的第一个和最后一个位置

第一次尝试

第一次做这个题，在二分法基础一般的情况下终于解决了……记录一下做题思路，目前的解决方案还可以优化。

  • 要实现第一个位置和最后一个位置，最重要的是先得到第一个比target小的数和第一个比target大的数，这两个位置分别加一减一就得到想要的位置坐标

  • 要注意考虑所给的情况：

    1. 所给值在范围之外，此时有一个边界并不会移动，这种情况下直接判断边界的初始值就行。以所给值小于范围为例：（表格第一行用于表示left,middle,right的位置；rb和lb表示边界，第二行是数组，第三行代表索引）

      假设target=0

      | | left（rb） | | middle | | | right（lb） |
      | :—: | :————: | :—: | :——: | :—: | :—: | :————-: |
      | | 1 | 2 | 2 | 2 | 2 | 5 |
      | | 0 | 1 | 2 | 3 | 4 | 5 |

      经过二分查找后最终会变成下面的样子：发现左边界并没有移动，易得如果target大于范围时右边界不移动。

      | right(lb) | left(rb) middle | | | | | |
      | :———-: | :———————: | :—: | :—: | :—: | :—: | :—: |
      | | 1 | 2 | 2 | 2 | 2 | 5 |
      | | 0 | 1 | 2 | 3 | 4 | 5 |

    2. 所给之虽然在范围之内，但是找不到该值,这种情况下两边界相邻

      假设target=3

      经过二分查找之后会变成下面的样子：

      | | | | | right(lb) | left(rb) middle |
      | :—: | :—: | :—: | :—: | :———-: | :——————-: |
      | 1 | 2 | 2 | 2 | 2 | 5 |
      | 0 | 1 | 2 | 3 | 4 | 5 |

    3. 在范围之内，正常考虑，左边界正好在第一个比target小的数，右边界正好在第一个比target大的数，二者差值大于1

    • 左右边界的循环条件是left<=right(左闭右闭法)，意味着终止条件是right跑到了left的左边或者left跑到了right的右边。
